# 修复项目编辑态 CLI 参数历史候选

## Goal

让编辑项目时的 CLI 启动参数输入框也能选择已保存的历史候选，并在编辑保存成功后累计历史使用次数。

## Root-Cause Statement

问题位于 `ConfigModal` 的表单与保存分支：编辑态被显式渲染为普通 `Field`，且编辑保存路径绕过历史记录；修复应落在这两个分支选择逻辑，而非历史存储层。

## Discovery List

- `src/components/ConfigModal.tsx`：编辑态候选渲染错误且保存未记录历史；需要修改。
- `src/lib/cliArgsHistory.ts`：按工具筛选、排序与上限正确；确认无关。
- `src/stores/settingsStore.ts`：历史读取与持久化接口正确；确认无关。
- `CHANGELOG.md`、`docs/功能清单.md`：当前文案称编辑不显示历史；需要修正。

## Requirements

- 新建和编辑项目在已选择 CLI 工具时，均展示同一历史候选下拉。
- 输入 CLI 参数时，从当前工具的全部历史中进行不区分大小写的包含匹配，匹配结果最多显示 10 条。
- 编辑保存成功后累计当前 CLI 参数的历史使用次数。
- 克隆项目继续不显示候选、不累计历史。

## Acceptance Criteria

- [ ] 编辑项目可展开当前 CLI 工具的前 10 条历史候选。
- [ ] 输入 `--yo` 可匹配当前工具全部历史中包含该片段的候选，且结果不超过 10 条。
- [ ] 选择候选后仍可自由修改参数并保存。
- [ ] 编辑保存后当前 CLI 参数的历史次数增加一次。
- [ ] 新建和克隆的既有计数规则不变。
- [ ] TypeScript 类型检查和专项测试通过。

## Scenario Matrix

| 场景 | 预期 |
|---|---|
| 新建项目 | 显示候选；成功创建后累计 |
| 编辑项目 | 显示候选；保存成功后累计 |
| 克隆项目 | 不显示候选；创建不累计 |
| 切换 CLI 工具 | 候选随工具切换 |
| 无历史 | 保持普通自由输入 |
| 本地 / WSL / SSH | 行为一致 |

## Technical Approach

- 将历史组合输入框的渲染条件从“仅空白新建”收窄为“非克隆”，并在新建/编辑共用的成功保存路径记录非空参数。

## Out of Scope

- 克隆保存时累计历史。
- 改变历史存储、同步和排序规则。
