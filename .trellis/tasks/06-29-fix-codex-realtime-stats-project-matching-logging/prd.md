# PRD

## 背景

实时统计面板在 Codex 项目中提示“该项目暂无 codex 会话记录”，但日志已确认 `/home/silver/.codex/sessions` 扫描成功且存在会话文件。

## 目标

1. 为实时统计的 Codex 会话定位链路补齐可追踪日志，覆盖前端请求、后端过滤、项目路径匹配、会话命中/未命中。
2. 修复 Codex 项目匹配过严导致的实时统计误判问题，使同一项目根目录与其子目录场景都能正确命中会话。

## 非目标

1. 不调整 Claude 历史扫描逻辑。
2. 不改动历史会话列表、分析看板等无关路径的展示逻辑。
3. 不新增设置项或调试开关。

## 验收

1. Debug 日志中可看到 `history_list_sessions` 请求参数、项目匹配决策、命中/未命中原因。
2. Codex 会话 `cwd` 为项目根目录、当前终端 `cwd` 为其子目录时，实时统计不再误报“暂无会话记录”。
3. `npx tsc --noEmit` 与 `cd src-tauri && cargo test` 至少覆盖本次改动相关验证，无法运行时需明确说明。
