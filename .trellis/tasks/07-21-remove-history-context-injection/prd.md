# 移除历史上下文注入并尊重用户项目分组

## Goal

移除 Claude/Codex 新会话的历史上下文文件生成与启动参数注入，并保证用户手动调整后的项目名称和分组在重启后保持不变。

## Background

- 当前会为同项目最近的同步会话生成 `.cli-manager/synced-history-context/*`，并通过 Claude `--append-system-prompt-file` 或 Codex `developer_instructions` 注入新会话。
- 启动时 `externalSessionSyncStore.load()` 异步重新物化已同步项目，会把用户移到根级并改名的项目再次改名为 `Claude`/`Codex` 并塞回同名分组。
- 用户明确要求：用户调整结果优先，不再生成或注入历史上下文文件。
- 用户明确要求：不检测、不清理旧版本已生成的上下文目录或文件，只保证后续不再创建和更新。
- Changelog Target: `V1.3.0`。

## Requirements

- 完整删除 Claude/Codex 自动历史上下文文件生成和启动命令注入链路。
- 新建项目终端、分屏终端、外部终端、同步历史终端均使用原始启动命令。
- 保留历史记录扫描、侧栏同步、历史查看以及显式 resume 功能。
- 启动加载已同步历史时，只补齐确实缺失的项目；不得重命名、移动或重新分组已有项目。
- 首次安装主动同步历史项目时，继续按现有规则创建初始分组和 CLI 子项目。
- 用户确认同步项目后，自动将项目候选 key 写入持久化忽略列表；升级前已有的同步记录在 Store 加载时补齐项目忽略 key，后续同步扫描不再展示这些项目。
- 用户可见行为变更写入 `CHANGELOG.md` 的 `V1.3.0`。
- 终端字体下拉框必须以同一套终端字体规范化规则比较当前值、内置选项和所有系统字体选项，不能把已安装且已选择的字体误标为“当前自定义（保留）”。
- 字体匹配必须兼容带空格、中文、逗号及其他标点的字体族名称，不允许针对 `Maple Mono` 写特例。
- 历史项目同步执行期间必须拒绝重复调用；按钮保持禁用，Store 入口同时防止界面重渲染前的连续点击触发第二次同步。

## Acceptance Criteria

- [x] 启动 Claude/Codex 时不再出现 `--append-system-prompt-file` 或 `developer_instructions` 注入参数。
- [x] 应用不再创建或更新 `synced-history-context` 文件。
- [x] 用户把历史导入项目移到根级并重命名后，重启应用仍保持该名称和层级。
- [x] 首次历史项目同步仍能创建项目，历史列表和显式恢复功能正常。
- [x] 已同步项目自动进入忽略列表，后续同步扫描不再显示；未同步项目仍可正常显示和忽略。
- [x] TypeScript 类型检查通过，相关回归测试通过。
- [x] 所有可用系统字体使用终端回退栈保存后仍匹配原字体选项；真正不存在的自定义字体才显示“当前自定义（保留）”。
- [x] `syncProjectCandidates` 执行期间的重复调用不会再次创建项目或写入同步状态，成功或失败后允许下一次同步。

## Out of Scope

- 不扫描、不删除旧版本已生成的 `.cli-manager/synced-history-context` 目录或文件。
- 不修改 Claude/Codex 原始历史记录。
- 不改变显式 `claude --resume`、`codex resume` 行为。

## Notes

- 不新增依赖，不修改数据库结构，不改变 Claude/Codex 历史源文件。
- 依据前端质量规约未由 AI 启动桌面应用；重启后的视觉层级与同步弹窗行为需人工桌面验收。
