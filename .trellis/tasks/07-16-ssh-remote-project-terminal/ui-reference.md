## 1. UI 原则

- 延续 CLI-Manager 现有设置、项目弹窗、暗色/亮色主题和 Lucide 图标体系。
- 项目仍是左侧导航主体；SSH 图标表达运行环境，不增加自动服务器树。
- 复杂连接参数使用左侧分区或折叠高级设置，基本连接流程保持短。
- 测试连接展示逐阶段诊断，不只返回“成功/失败”。
- 所有禁用能力都说明原因和后续边界，不让用户误以为按钮失效。

## 2. 页面清单

### U1 SSH 主机列表

- 设置导航中的“SSH 主机”；
- 分组树/筛选、搜索、连接状态；
- 新建、复制、编辑、删除、测试、打开终端；
- 空状态与 OpenSSH 未安装状态。

### U2 新建/编辑 SSH 主机

参考 XTerminal 的字段覆盖，但重新组织为：

1. 基本信息；
2. 认证；
3. 跳板与代理；
4. 连接设置；
5. 初始化。

底部固定“测试连接 / 取消 / 保存”。密码和秘密字段需要明确的存储状态标识。

### U3 测试连接诊断

分阶段显示：OpenSSH → 网络 → 主机指纹 → 认证 → Shell → SFTP。允许复制脱敏诊断。

### U4 新建 SSH 远程项目

- 沿用现有“新增终端”窄幅单列表单；
- 标题下方增加“本地项目 / SSH 远程项目”Tab，作为两套表单的一级切换；
- 本地项目 Tab 对应现有表单，SSH 远程项目 Tab 对应新的纵向远程表单；
- 选择主机；
- 远程路径输入、浏览和检测；
- 项目分组、CLI、启动参数；
- 能力提示：首期文件/Git/Worktree/历史不可用。

本地项目保持现状；SSH 状态隐藏 Shell 和 Worktree 等不适用字段，并以能力提示说明首期边界。

### U5 远程目录浏览器

- 面包屑、返回上级、刷新、显示隐藏目录；
- 目录列表、加载/认证/错误/无 SFTP 状态；
- 手工路径输入与检测降级。

### U6 项目树与终端状态

- SSH 项目徽标；hover 展示主机与远程路径；
- connecting/authenticating/connected/disconnected 状态；
- 右键菜单按能力过滤。

### U7 不支持能力说明

文件、Git、Worktree、历史等入口使用统一远程能力空态，而不是散落 toast。

## 3. 必须出图的关键页面

1. SSH 主机列表；
2. 新建/编辑 SSH 主机——基本信息与认证；
3. 新建/编辑 SSH 主机——跳板、代理与连接设置；
4. 测试连接诊断；
5. 新建 SSH 远程项目；
6. 远程目录选择器；
7. SSH 项目在项目树、Tab 和断线状态中的组合视图；
8. 远程能力未支持空态。

每张图至少提供暗色主题；关键表单补充亮色主题检查。所有文案同时给出 zh-CN/en-US 对照表。

## 4. XTerminal 参考取舍

保留：字段完整性、左侧配置分区、底部测试连接、认证顺序、跳板和代理独立页面。

调整：CLI-Manager 不把连接弹窗做成运维工具控制台；默认只展示常用字段，高级参数渐进展开；项目创建与主机配置分离。
