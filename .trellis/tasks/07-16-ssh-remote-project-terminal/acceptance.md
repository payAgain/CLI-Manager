# SSH 远程项目 / 远程终端逐项验收

## 1. 验收说明

- 验收分支：`feat/ssh-remote-project-terminal`
- 关联 Issue：#145
- 首期范围：SSH 主机管理、远程项目创建、远程目录浏览/检测、SSH PTY 会话、连接状态、会话恢复和不支持能力降级。
- 首期不包含：远程文件管理、远程 Git、远程 Worktree、远程历史/Hook/统计、远程 Provider 切换和远程系统资源监控。
- 自动验证不代替真实服务器、首次主机指纹、密码/MFA、跳板机和网络中断等人工验收。

建议准备以下环境：

1. 一台可直连的 Linux SSH 主机。
2. 可用的 SSH Agent 或私钥认证配置。
3. 可选的密码/MFA 主机。
4. 可选的跳板机或 `ProxyCommand` 配置。
5. 一个包含空格和 Unicode 字符的远程目录。

每项完成后在“实际结果 / 备注”中填写 `通过`、`失败` 或问题说明。

## 2. SSH 客户端与主机管理

### SSH-01 OpenSSH 探测

- 前置条件：分别在系统 OpenSSH 可用和不可用的环境验证。
- 操作：打开“设置 → SSH 主机”。
- 预期：可用时正常进入；不可用时显示明确的中英文安装/配置提示，不影响本地项目。
- 实际结果 / 备注：

### SSH-02 主机 CRUD 与多级分组

- 操作：新增根分组、子分组；新增主机时通过分组下拉框选择分组；随后编辑、复制并删除副本和分组。
- 预期：SSH 主机列表按多级树展示；新增、编辑、复制、删除均正确；删除分组时子分组和主机提升到父级；项目树不会自动生成 SSH 主机分组。
- 实际结果 / 备注：

### SSH-03 SSH Config / Agent 认证

- 操作：分别创建使用 SSH Config alias 和 SSH Agent 的主机并测试连接。
- 预期：复用系统 OpenSSH 配置和 Agent；测试结果能区分客户端探测、网络、认证和远端命令阶段。
- 实际结果 / 备注：

### SSH-04 私钥认证

- 操作：选择本机私钥文件，连接使用该私钥的主机。
- 预期：连接成功；私钥内容和口令不写入数据库、日志或同步数据；私钥路径仅作为本机配置使用。
- 实际结果 / 备注：

### SSH-05 用户/密码、每次询问与交互认证

- 操作：分别使用密码询问、每次询问、键盘交互/MFA 主机测试连接并打开终端。
- 预期：“用户/密码”保存到系统凭据库并可用于测试连接、路径检测和目录浏览；密码不进入 SQLite、日志、同步和命令行；“每次询问”和键盘交互/MFA 仍在终端 PTY 中人工完成，无法非交互目录浏览时允许手工路径降级。
- 实际结果 / 备注：

### SSH-06 首次指纹与指纹变化

- 操作：首次连接新主机；随后在测试环境模拟 Host Key 变化。
- 预期：首次指纹由 OpenSSH 明确询问；指纹变化时强制阻断并显示原因，不提供默认忽略继续的动作。
- 实际结果 / 备注：

### SSH-07 跳板机

- 操作：配置引用现有 SSH 主机的跳板机，覆盖非默认用户名/端口；如可用，再验证 IPv6 地址。
- 预期：连接经过正确的跳板参数；禁止形成循环引用；错误能定位到跳板链路。
- 实际结果 / 备注：

### SSH-08 ProxyCommand / 代理安全

- 操作：配置合法 `ProxyCommand` 并测试；尝试保存 URL 内嵌用户名或密码的代理配置。
- 预期：合法配置可用；内嵌凭据被拒绝；日志不会输出完整敏感代理参数。
- 实际结果 / 备注：

### SSH-09 超时与 KeepAlive

- 操作：连接不可达地址并设置较短超时；再验证 KeepAlive 参数主机。
- 预期：超时后结束尝试并显示可理解错误；界面不会永久停留在连接中。

## 6. 状态、断线与恢复

### SSH-21 连接状态展示

- 操作：观察连接、认证、成功、失败和断开阶段的 Tab、悬浮信息及项目位置。
- 预期：状态不只依赖颜色表达；主机、远程路径和断开原因可识别；后台任务显示远程位置。
- 实际结果 / 备注：

### SSH-22 断开原因分类

- 操作：分别让远端 Shell 正常退出、CLI 命令退出、网络中断以及本地 SSH 进程异常退出。
- 预期：分别归类为远端退出、远端命令退出、SSH 传输错误和本地进程错误，不统一显示为未知失败。
- 实际结果 / 备注：

### SSH-23 daemon 存活会话恢复

- 操作：保持 SSH 会话运行，退出并重新打开 CLI-Manager。
- 预期：重新 attach daemon 中的原 PTY，保留输出与交互；不会创建第二条 SSH 连接或重复运行启动命令。
- 实际结果 / 备注：

### SSH-24 已退出会话恢复

- 操作：让 SSH 会话退出后关闭并重开应用。
- 预期：只恢复回放/断开状态，不重新执行 CLI、初始化脚本或其他远程命令。
- 实际结果 / 备注：

### SSH-25 旧 daemon 协议兼容

- 操作：在可复现环境使用不支持 SSH 创建帧的旧 daemon 启动 SSH 会话。
- 预期：应用安全降级到进程内 PTY；本地终端旧 Create 帧仍可正常工作。
