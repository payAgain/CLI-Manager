# SSH 集成场景矩阵

## 1. 连接与认证

| 维度 | 场景 | 期望行为 |
|---|---|---|
| SSH 客户端 | 已安装 / 未安装 / 版本过低 | 启动前探测；不可用时阻止连接并给出平台化安装提示 |
| 主机指纹 | 已信任 / 首次连接 / 指纹变化 | 首次由明确确认 UI 或原生 SSH 交互处理；变化必须高危阻断，不自动接受 |
| 认证 | Agent / 私钥 / 密码 / MFA / 每次询问 | 都允许系统 SSH 交互；敏感输入不进入日志和历史 |
| SSH Config | Host alias / Include / ProxyJump / Match | 优先交给系统 SSH 解析；CLI-Manager 自有字段只生成显式覆盖参数 |
| 网络 | 正常 / DNS 失败 / 超时 / 断网 / VPN 变化 | 分类展示错误；允许重试；不把网络失败标为项目路径丢失 |
| 跳板与代理 | 单跳 / 多跳 / HTTP/SOCKS / 配置引用 | MVP 优先支持 Host alias/ProxyJump；自定义代理能力分期实现 |

## 2. 项目与路径

| 场景 | 期望行为 |
|---|---|
| 一台主机多个项目 | 多个项目引用同一 `ssh_host_id`，各自保存远程目录与项目配置 |
| 同一路径不同主机 | 视为不同项目运行环境，不因路径字符串相同合并 |
| 远程目录不存在 | 项目保留并标为不可达/路径异常；用户可修改或重试，不自动删除 |
| SFTP 禁用 | 允许手工输入路径并通过远程 shell 检测 |
| 非 POSIX Shell | MVP 明确以远程 Linux/POSIX Shell 为正式支持目标 |
| 路径含空格/引号/Unicode | 使用参数化 Launch Plan 和严格 shell 转义测试 |

## 3. CLI-Manager 运行状态

| 维度 | 场景 | 期望行为 |
|---|---|---|
| 窗口 | 当前窗口 / 其他窗口 / 最小化 / 托盘 | SSH 会话与本地会话使用同一 Tab/Pane 生命周期 |
| 分屏 | 当前 Pane / 其他 Pane / 深层分屏 | 新 SSH Tab 和分屏遵守现有 PaneTree，不重建 xterm |
| Workspan | 单工作区 / 多工作区 / 项目筛选 | 通过 `projectId` 归属，不按 cwd 猜测主机 |
| 后台任务 | UI 退出 / daemon attach / 网络断开 | 保留本地 SSH PTY；远端任务持久性不做虚假承诺，建议后续 tmux 模式 |
| 恢复 | SSH 进程仍存活 / 已退出 / 网络变化 | attach 存活会话；已退出显示断开状态，不自动重复执行启动命令 |

## 5. 安全场景

- 密码、私钥口令、Token 不写 SQLite、Store、日志、错误详情或会话快照。
- 私钥路径属于机器本地配置；同步时默认剔除或标记需重新绑定。
- 远程命令由固定模板和参数生成，禁止拼接未经转义的项目名、路径或备注。
- Host Key 变化不得提供“一键忽略并继续”的默认动作。
- 跳板链与代理日志只能记录非敏感拓扑，不记录代理 URL 中的凭据。
