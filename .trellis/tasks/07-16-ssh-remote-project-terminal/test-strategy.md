# SSH 集成测试与验收策略

## 1. 自动化测试

- 数据迁移：旧项目全部保持 local；SSH 主机删除/引用约束正确。
- SSH 参数生成：Host alias、端口、用户、IdentityFile、ProxyJump、超时和 KeepAlive。
- POSIX 路径与启动命令转义：空格、引号、Unicode、换行和恶意输入。
- 能力路由：SSH 项目不得调用本地 fs/git/worktree/history 路径命令。
- 日志脱敏：密码、口令、代理凭据和秘密环境变量不会出现。
- 会话持久化：SSH 元数据恢复、断开状态和项目归属正确。

## 2. 集成环境

- Windows OpenSSH → Linux；macOS → Linux；Linux → Linux。
- 直连、SSH Config alias、Agent、私钥、密码、MFA、ProxyJump。
- SFTP 开启/关闭、远端目录存在/不存在/无权限。
- 网络中断、重连、主机重启、Host Key 变化。

## 3. 全应用回归

- 本地和 WSL 项目创建、启动、分屏、恢复行为完全不变。
- SSH 项目参与手动分组、排序、项目筛选和 Workspan。
- 文件/Git/Worktree/历史/Hook/统计入口不会误用本地路径。
- daemon 后台托管与退出弹窗能正确识别 SSH PTY。
- WebDAV/导出不包含秘密，其他设备导入后提示重新绑定本机凭据/密钥路径。

## 4. UI 与可访问性

- zh-CN/en-US 完整；英文时间保持 24 小时制。
- 键盘可完成主机创建、测试连接、目录选择和项目创建。
- 错误与连接状态不能只依赖颜色。
- 暗色/亮色均满足对比度，窄窗口无横向溢出。

## 5. MVP 验收主流程

1. 新建 SSH 主机并完成测试连接；
2. 使用该主机浏览或填写远程目录并创建项目；
3. 从项目树打开远程终端，正确进入目录并启动指定 CLI；
4. 在分屏和 Workspan 中运行多个本地/远程会话；
5. 断网后显示断开原因，恢复时不重复执行危险命令；
6. 尝试文件/Git/Worktree/历史入口时得到一致、明确的能力说明；
7. 退出应用并恢复 daemon 托管的 SSH 会话；
8. 检查日志、数据库、导出和同步文件无秘密泄漏。
