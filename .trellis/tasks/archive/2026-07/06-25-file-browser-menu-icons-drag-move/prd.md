# 文件浏览器菜单图标与拖拽移动

## Goal

优化文件浏览器右键菜单与移动交互：给常用菜单补图标，根目录支持粘贴，并将文件/文件夹移动从右键菜单改为拖拽到目标目录。

## What I already know

* 文件浏览器菜单实现在 `src/components/files/FileExplorerSidebar.tsx`。
* 当前 `重命名`、`复制`、`移动` 右键菜单项没有图标。
* 当前移动逻辑通过 `clipboard.mode === "move"` 后调用 `file_move` 实现。
* 后端已有 `file_move` 命令，拖拽移动可复用现有移动能力，不需要新增后端命令。
* 当前文件树拖拽已用于向终端插入路径，尚未实现树内拖拽移动。

## Requirements

* 给文件树右键菜单里的 `重命名`、`复制` 增加图标。
* 根目录右键菜单增加 `粘贴`，用于把复制项粘贴到项目根目录。
* 删除文件/文件夹右键菜单里的 `移动` 菜单项。
* 支持文件/文件夹拖拽移动到目标目录。
* 保留现有复制、粘贴、重命名、删除、AI 路径复制行为。

## Acceptance Criteria

* [ ] 文件/文件夹右键菜单中 `重命名`、`复制` 显示图标。
* [ ] 文件树空白/根目录右键菜单在剪贴板有内容时可点击 `粘贴`。
* [ ] 文件/文件夹右键菜单不再显示 `移动`。
* [ ] 将文件拖到目录上会移动到该目录。
* [ ] 将文件拖到文件上会移动到该文件所在目录。
* [ ] 将文件拖到根目录空白区域会移动到项目根目录。
* [ ] 拖拽移动遇到重名目标时沿用现有覆盖确认。
* [ ] TypeScript 检查通过。

## Definition of Done

* 代码改动最小，优先复用现有 `pasteInto` / `file_move` 行为。
* 不新增依赖。
* 不改后端接口，除非前端无法复用现有能力。
* 完成 `npx tsc --noEmit` 验证。

## Out of Scope

* 不实现多选拖拽。
* 不实现跨项目拖拽移动。
* 不调整文件树整体样式。
* 不新增复杂拖拽动画。

## Technical Notes

* 相关文件：`src/components/files/FileExplorerSidebar.tsx`。
* 图标从 `src/components/icons.ts` 导出的 lucide 图标复用。
* 拖拽移动应避免把目录移动进自身或子目录。
