# 优化历史会话请求日志界面

## Goal

在不改变请求日志数据口径、查询参数和后端接口的前提下，将请求日志页调整为适合高密度日志检索的紧凑布局，提升筛选效率、表格扫描性、异常识别和会话跳转体验。

## Requirements

- 完成已确认的 P0、P1、P2 全部优化。
- 筛选区分为主要条件与可展开的高级条件，保留来源、项目、模型、会话、开始日期、结束日期及查询、重置、同步刷新能力。
- 日期提供今天、近 7 天、近 30 天快捷范围；快捷范围只修改筛选草稿，查询行为保持可控。
- 汇总卡片压缩为紧凑指标条，继续展示记录数、Token 总量、估算费用和未定价 Token。
- 表头吸顶，数字列右对齐并使用等宽数字；行提供稳定 hover 高亮和轻量隔行底色。
- 项目名称保持主要信息，会话 ID 截断展示并支持复制完整值。
- 输入 Token 以输入、缓存读取、缓存写入的紧凑结构展示；输出与费用保持独立数字列。
- 来源、模型和状态标签支持点击后写入对应筛选草稿；状态列避免为所有正常行重复显示高强调标签。
- 高 Token、高费用和未定价记录提供轻量异常提示，不改变统计数据。
- 会话可用时支持整行双击打开；操作列保留紧凑图标按钮并提供可访问名称。
- 保持水平滚动、小窗口布局、键盘焦点、加载/错误/空状态和分页可用。
- 所有新增用户可见文案同步支持 `zh-CN` 与 `en-US`，时间继续使用 24 小时制。
- Changelog Target：`[TEMP]`。

- [x] 今天、近 7 天、近 30 天快捷日期可正确更新日期范围。

- [x] 点击来源、项目或模型标签可快速应用对应筛选条件；状态为单一后端值，仅弱化展示，不增加无意义筛选。

## Scenario Matrix

- 窗口焦点：当前窗口、其他窗口、应用失焦均不改变筛选和日志打开语义。
- 分屏/多会话/Workspan：请求日志页只调用既有 `onOpenSession`，不新增终端或改变会话归属。
- 最小化/托盘：不新增后台行为；继续依赖现有定时同步与手动刷新。
- 运行环境：Claude/Codex、本地/WSL/Bash 记录继续按后端返回数据展示，不改路径或来源解析。
- Worktree：项目名、会话 ID、文件路径继续使用现有日志字段，不新增推断。
- Hook 安装状态：页面读取本地请求日志；未安装或仅安装一种 Hook 时仍能筛选现有来源数据。

## Technical Approach

- 主要修改 `src/components/stats/RequestLogsView.tsx`，仅做前端展示和本地交互调整。
- 在 `src/lib/i18n.ts` 补齐快捷日期、高级筛选、复制反馈和可访问性文案。
- 更新 `CHANGELOG.md` 的 `[TEMP]` 与 `docs/功能清单.md`，记录用户可见行为变化。
- 不修改 Rust 请求日志接口、数据库结构、分页大小、费用计算和同步流程。
- 使用现有 Tailwind、Lucide、Button、Card、Select、StatsDatePicker，不新增依赖。
