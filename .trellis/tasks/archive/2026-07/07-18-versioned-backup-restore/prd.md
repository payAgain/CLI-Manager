# 版本化项目与设置备份恢复

## Goal

将现有覆盖式 WebDAV“同步”重构为版本化备份与手动恢复：主要保护项目配置和便携设置，退出时仅在内容变化后创建备份，支持查看历史快照、按域完整恢复与失败回滚。

## Requirements

- 备份数据域固定为：工作区配置、应用偏好、模型价格、三方 Hook 通知、状态栏配置。
- 工作区配置包含分组、项目、active Worktree、持久命令模板以及项目完整字段和时间戳。
- 应用偏好只包含跨设备便携设置；设备路径、Shell Profile、WSL/显卡开关、运行状态、资源文件和派生数据不得进入备份。
- 状态栏备份包含 CLI-Manager 管理的当前配置与多配置库，但恢复不得自动修改外部 Claude/Codex 安装状态。
- 快照保持明文，不引入加密；界面必须明确披露环境变量、命令提示 API Key、命令模板和三方 Hook 凭据会进入快照。
- WebDAV 快照不可变且可枚举、下载、删除；每个设备只保留最近 10 份。
- 本地备份继续使用 ZIP；WebDAV 使用版本化 JSON 快照。
- 退出时先写本地 outbox，再在既有退出超时内上传；失败或超时后下次启动异步重试。
- 自动备份按规范化内容哈希去重；手动备份始终创建新快照。
- 恢复按域完全还原，执行前创建本地安全快照，失败自动回滚并支持撤销最近一次成功恢复。
- 项目路径按原值恢复；本机不存在时沿用项目健康检查标记不可用。Worktree 缺失路径标记为 missing。
- 保留 V1/V2 WebDAV JSON、旧 sync.json 和旧本地 ZIP 的显式导入兼容。
- 移除启动自动下载与冲突处理；旧退出 upload 设置迁移为退出自动备份。
- 不备份终端标签/分屏/Workspan、背景图、桌宠资源、会话历史、命令历史、缓存、日志、外部 Claude/Codex/cc-switch 数据和系统凭据。

## Acceptance Criteria

- [ ] 手动和退出自动备份均生成 V3 快照，自动备份无变化时跳过。
- [ ] WebDAV 可列出版本快照并按设备保留最近 10 份；清理失败不影响新快照成功。
- [ ] 网络失败或退出超时后 outbox 保留，下一次启动可重试上传。
- [ ] 五个数据域可预览并按选择完整恢复，未选择域保持不变。
- [ ] 恢复前安全快照可用于自动回滚和一次撤销。
- [ ] 设备设置、运行状态、资源文件和外部工具数据不进入快照。
- [ ] 状态栏文件恢复经过现有校验并原子写入，不触碰外部安装配置。
- [ ] V1/V2 JSON 和旧 ZIP 仍可导入。
- [ ] 中英文文案、明文风险提示和 24 小时时间格式正确。
- [ ] `npx tsc --noEmit`、`cargo check --locked --manifest-path src-tauri/Cargo.toml`、相关 Rust 测试通过。
