# ccusage 看板全屏 UI 设计

## 设计约束

- 只做全屏布局和视觉层级调整，不新增任何功能。
- 保留现有来源选择、刷新/工具准备、时间窗口和错误/空状态逻辑。
- 保留现有 4 个 KPI、统计口径、峰值日摘要、Token 构成、趋势、热点图、模型排行和数据结构摘要。
- 沿用历史用量分析全屏稿的顶栏、卡片、间距、圆角和蓝色主色，保证两个看板风格统一。

## 全屏布局

1. 顶部工作区栏：返回、标题、来源摘要、缓存更新时间、关闭。
2. 筛选上下文栏：数据来源、时间窗口、刷新、工具状态、当前记录数和统计口径。
3. KPI 行：总 Token、估算费用、最高使用日、模型/天数。
4. 中间主区域：单独占满整行的 Token / 费用趋势。
5. 底部四列：峰值日摘要、Token 构成、Token 热点图、模型用量排行。
6. 底部信息栏：数据结构摘要与 payload 字段。

## 功能映射

| 现有功能 | 全屏稿位置 | 是否变化 |
|---|---|---|
| 来源选择 | 顶部筛选栏 | 否 |
| 刷新报告 | 顶部筛选栏 | 否 |
| 工具准备/安装 | 原逻辑保留，异常时显示 | 否 |
| 时间窗口 | 顶部筛选栏 | 否 |
| KPI | 首屏指标行 | 否 |
| 统计口径 | 筛选栏下方 | 否 |
| Token / 费用趋势 | 中间全宽区域 | 仅放大 |
| 峰值日摘要 | 底部第一列 | 仅重排 |
| Token 构成 | 底部第二列 | 仅重排 |
| Token 热点图 | 底部第三列 | 仅重排 |
| 模型用量排行 | 底部第四列 | 仅重排 |
| 数据结构摘要 | 页面底部 | 否 |
