# 历史用量分析全屏 UI 设计

## 设计方向

- 从居中弹窗改为占满应用内容区的独立工作区，移除黑色遮罩与固定 `max-w-6xl` 限制。
- 使用“用量概览 / 请求日志”两个一级页签，共用项目、来源与时间范围上下文。
- 延续现有浅色主题、蓝色主色、圆角卡片与 Recharts 配色，不引入新设计体系。
- 首屏优先展示可行动信息：总量、费用、请求数、峰值、计价覆盖、趋势、构成、项目/模型排行和活跃时段。

## 页面结构

1. 顶部工作区栏：返回、标题、数据范围、刷新时间、关闭。
2. 一级页签：用量概览、请求日志；日志页签显示当前记录数。
3. 共用筛选区：项目、来源、时间范围；请求日志增加供应方、模型/会话搜索和状态筛选。
4. 用量概览：5 个 KPI + 单独占满中间区域的主趋势图 + 下方 4 个辅助分析卡片，Token 构成位于最后一列。
5. 请求日志：5 个摘要指标 + 高密度明细表 + 分页。

## 请求日志字段映射

| 界面字段 | 现有数据可用性 | 设计处理 |
|---|---|---|
| 时间 | 可用 | usage 事件时间 |
| 供应方 | 部分可用 | 暂显示 Claude / Codex Session |
| 计费模型 | 可用 | usage 事件模型 |
| 项目 / 会话 | 可用 | 从所属历史会话补充 |
| 输入 / 输出 / 缓存 | 可用 | usage 事件 Token 构成 |
| 总成本 | 可用 | 本地价格估算 |
| 耗时 | 不可用 | 暂显示 `—` |
| 状态 | 无 HTTP 状态 | 使用“已解析 / 未计价”等本地状态 |
| 来源 | 可用 | `codex_session` / `claude_session` |
