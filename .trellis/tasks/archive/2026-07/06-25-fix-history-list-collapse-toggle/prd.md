# 修复历史列表折叠无法关闭

## Goal

修复历史会话面板里项目来源分组展开后无法再次折叠的问题，保持现有历史列表、搜索、项目过滤行为不变。

## Requirements

* 历史会话面板顶部“项目来源”下拉打开时，项目组默认折叠。
* 用户点击项目组后可以展开，再次点击可以折叠。
* 项目来源搜索时仍保持匹配分组展开，便于看到搜索结果。
* 历史详情原文消息的顶部角色栏可以展开/折叠自动折叠内容。
* 历史详情原文消息顶部右侧显示折叠状态图标，区分当前是展开还是收起。
* 不改动历史会话加载、搜索、删除、打开会话逻辑。

## Acceptance Criteria

* [ ] 打开“项目来源”下拉后，点击一个分组能展开。
* [ ] 再次点击同一个分组能关闭。
* [ ] 输入项目来源搜索词时，匹配分组仍自动展开。
* [ ] 点击自动折叠消息顶部角色栏可以展开，再次点击可以收起。
* [ ] 自动折叠消息顶部右侧有明确的展开/收起图标。
* [ ] `npx tsc --noEmit` 通过。

## Definition of Done

* 只做必要的前端状态修复。
* 静态类型检查通过。
* 说明需要人工在桌面 UI 中验证的交互。

## Technical Approach

限制“打开下拉时默认折叠全部项目组”的重置时机，避免每次项目树引用变化或交互重渲染都覆盖用户的展开/折叠选择。

## Out of Scope

* 不新增会话时间分组折叠功能。
* 不调整历史会话列表虚拟滚动。
* 不改项目树数据结构或持久化设置。

## Technical Notes

* 主要文件：`src/components/history/HistoryListPane.tsx`
* `HistoryListPane` 影响分析：LOW，GitNexus 未发现上游受影响调用链。
* 定位结果：会话时间分组是静态标签；实际折叠状态在“项目来源”下拉树的 `collapsedFilterGroups`。
