# 终端重构验证清单

## 范围

本清单用于验证 XTerm 终端子系统重构。请在当前分支正常启动桌面应用后执行。

已确认存在于正式版本、且不作为本次验收阻塞项的问题见 `docs/terminal-known-issues.md`。

## 验证记录

| 项目 | 结果 | 备注 |
| --- | --- | --- |
| 环境 | 通过 / 失败 | Windows 版本、Shell、CLI 版本 |
| PowerShell | 通过 / 失败 | |
| Claude Code | 通过 / 失败 | |
| Codex | 通过 / 失败 | |
| CMD 或 Git Bash | 通过 / 失败 | |

## 1. 终端生命周期

- [ ] 新建 PowerShell 终端，确认提示符出现且可以输入。
- [ ] 分别新建 Claude Code 与 Codex 终端，确认均能进入正常交互界面。
- [ ] 每个终端都有近期输出时反复切换标签，确认终端不重启且不丢失输出。
- [ ] 关闭一个终端，为同一项目重新打开终端，确认其他标签仍可正常使用。
- [ ] 在存在活动会话时重启应用，确认 Shell 滚动历史与 TUI 会话遵循既有恢复行为。

## 2. OSC 与会话状态

- [ ] 在 PowerShell 执行切换目录命令，确认显示的会话工作目录更新。
- [ ] 通过 Shell 集成执行切换目录命令，切换标签后项目/会话路径仍正确。
- [ ] 确认 Claude 与 Codex 的状态指示器、终端颜色仍正常显示。
- [ ] 在浅色与深色终端主题下，确认 OSC 触发的颜色回复不会显示为转义字符文本。

## 3. 显示、缩放与渲染器

- [ ] 切换深色和浅色终端主题，确认文本、光标、选区与搜索高亮均清晰可读。
- [ ] 启用和关闭终端背景图，确认透明度、模糊、压暗、填充方式与位置正确。
- [ ] 在非透明深色主题下启用硬件加速，确认终端正常渲染。
- [ ] 启用低内存模式，隐藏标签超过十秒后返回，确认会话仍存活、可重绘且不丢失滚动历史。
- [ ] 横向和纵向调整应用窗口大小，确认终端适配新视口且 Shell 仍可交互。
- [ ] 在终端上按住 Ctrl 滚动鼠标滚轮，确认字号变化会持久化；普通滚动仍正常。
- [ ] 终端位于底部和查看历史位置时，分别打开、关闭右侧面板；发生跳动时按已知问题文档记录，不作为本次重构验收失败。

## 4. PTY 输出与可见性

- [ ] 执行持续输出命令，切换到其他标签数秒后返回，确认近期输出完整且顺序正确。
- [ ] 在 Claude 与 Codex 流式输出时重复上一项验证。
- [ ] 调整窗口大小时在可见和隐藏标签间切换，确认不会出现空白终端或永久遮罩。
- [ ] 无隐藏期输出时返回标签，确认终端仍可交互且尺寸适配正确。

## 5. 输入编辑与命令提示

- [ ] 输入 ASCII 文本，使用左右方向键移动、在中间插入，再使用 Backspace 与 Delete，确认 Shell 接收到预期命令。
- [ ] 输入中文等 CJK 文本，使用方向键移动并编辑，确认光标移动和删除逻辑正确。
- [ ] 向 PowerShell 粘贴单行和多行文本，确认多行文本顺序正确且不会逐行提交。
- [ ] 向 Claude Code 粘贴多行文本，确认保持为一次粘贴操作。
- [ ] 使用 Enter 提交命令，确认命令可以执行；切换目录命令应更新会话路径。
- [ ] 验证手动 Codex Enter 路径与配置的终端换行快捷键。
- [ ] 完成输入、退格、左右移动和粘贴后，确认模板、内置命令、路径与 AI 命令建议仍会刷新，并可通过 Tab、右方向键和 Ctrl+Space 接受。

## 6. 选择与复制

- [ ] 使用鼠标选择文本后按 Ctrl+C 复制，确认终端进程不会被中断。
- [ ] 无选区时运行长命令并按 Ctrl+C，确认命令被中断。
- [ ] 对新输入文本使用 Shift+左方向键和 Shift+右方向键选择，再用 Backspace 与 Delete 删除选区。
- [ ] 右键已有选区，确认会复制并清除选区；无选区右键应打开终端菜单。
- [ ] 验证普通点击聚焦和鼠标文本选择。点击不得改变终端光标位置，点击定位功能已明确不支持。

## 7. 输入法

- [ ] 在 PowerShell 中使用输入法输入中文，包括标点和候选词确认。
- [ ] 在 Claude Code 和 Codex 中重复输入法验证。
- [ ] 组字期间调整一次应用窗口大小，确认候选框保持在当前输入附近，而非跳到左上角。
- [ ] 输入法确认后，确认文本只提交一次，且命令建议反映已提交文本。
- [ ] 组字期间或刚结束后切换会话，确认前一个终端的文本不会写入新终端。

## 8. 粘贴与文件拖放

- [ ] 从浏览器剪贴板粘贴纯文本。
- [ ] 向项目终端粘贴截图，确认插入附件路径且文件存在于项目附件目录。
- [ ] 从应用内文件浏览器将文件拖入 PowerShell、Claude 和 Codex 终端，确认使用对应 Shell 的路径引用方式。
- [ ] 从 Windows 资源管理器将文件拖入当前可见终端，确认只插入一次。
- [ ] 将文件拖放到非活动或隐藏终端，确认不会插入错误会话。

## 9. 右键菜单与搜索

- [ ] 打开终端搜索，查找多个匹配项、切换匹配项并关闭搜索。
- [ ] 搜索打开期间，确认匹配装饰仍可见；关闭后恢复正常终端选择。
- [ ] 验证当前布局可用的右键菜单操作：复制、粘贴、全选、复制全部、清屏、分屏、关闭及会话管理。

## 10. 后台任务恢复

- [ ] 启动 Claude Code 或 Codex 任务，在退出确认中选择“退出应用，任务在后台继续执行”。
- [ ] 重新打开应用，在右侧后台任务中心点击恢复；确认新 Workspan/Tab 打开并显示 daemon 回放输出。
- [ ] 恢复后继续与 Claude/Codex 交互，确认新输出持续出现，输入可正常发送，原任务没有重启或重复 resume。
- [ ] 确认恢复后的 Claude/Codex Tab 显示正确 CLI 图标，不是普通终端图标。
- [ ] 在后台任务持续输出期间点击恢复，确认回放尾部与恢复期间的新输出均存在、顺序正确、没有明显重复块。
- [ ] 让后台任务完成后再恢复，确认最终输出可查看，Tab 状态为已完成且仍显示正确 CLI 图标。
- [ ] 直接重启应用并恢复已有 daemon 会话，重复验证回放、新输出与 CLI 图标；该入口不得只在后台任务中心恢复时正常。

## 11. 命令历史入口调整

- [ ] 确认终端右侧工具栏不再显示“命令历史”按钮。
- [ ] 打开“设置 -> 侧边栏 -> 终端工具栏”，确认不再显示“历史命令”开关，也不能拖动该旧按钮。
- [ ] 打开“设置 -> 命令提示”，确认总开关、历史存储数量、占用空间和清理按钮均保留且可用。
- [ ] 记录当前历史存储数量，连续执行多条新命令后刷新统计，确认数量不增加。
- [ ] 使用清理前已有的历史记录触发候选，确认历史建议仍正常；清理历史后确认统计归零。
- [ ] 确认命令模板、历史、内置 AI CLI 命令、路径补全和已配置的 AI 提示全部正常。
- [ ] 使用包含旧 `commandHistory` 工具栏配置的升级安装验证，确认应用正常启动，工具栏旧配置被忽略且其他工具栏顺序保持相对稳定。

## 12. 标签切换焦点

- [ ] 在两个普通终端 Tab 间通过鼠标切换，切换后直接输入，无需再次点击终端。
- [ ] 使用快捷键或鼠标侧键切换终端 Tab，切换后直接输入，字符只进入当前活动终端。
- [ ] 在多个 Workspan 间切换，切换后直接输入，当前 Workspan 的活动终端自动获得焦点。
- [ ] 在分屏中切换不同 Pane 的 Tab，确认只有全局活动且可见的终端获得焦点；可见但未激活的另一 Pane 不抢输入。
- [ ] 从历史会话页面返回终端、退出 Pane 全屏以及关闭右侧面板后，再切换 Tab，确认仍可直接输入。
- [ ] 在 Claude Code 与 Codex 中重复切换，确认 IME、光标和键盘输入均绑定到当前 Tab。

## 13. 完成条件

仅在满足以下条件时将本次重构标记为可提交：

- 以上适用项目至少在 PowerShell、Claude Code 和 Codex 中均通过。
- 相比正式版本未发现新的回归。
- 已复现的正式版本问题已在 `docs/terminal-known-issues.md` 补充环境与复现信息，或关联到该文档。
