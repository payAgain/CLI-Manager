# CLI-Manager 功能清单

> 更新时间：2026-07-20（覆盖至 V1.3.0 及 TEMP）
> 参考来源：`README.md`、`CHANGELOG.md`、当前前端与 Tauri/Rust 后端源码只读梳理。

## [TEMP]

### 终端文件路径跳转

- 终端工具栏“文件”开启时，Ctrl+单击项目或 Worktree 内的相对文件、目录路径会打开右侧文件面板；文件树自动展开父目录并选中目标目录。
- 文件路径支持 `:行`、`:行:列` 和 `(行,列)`，打开编辑器后会定位、选中并高亮目标行；绝对路径仍按原有逻辑打开系统资源管理器。

## [V1.3.0] - 2026-07-20

### SSH Config 导入

- SSH 主机设置支持从原生 Windows、Linux、macOS 用户的默认 `~/.ssh/config` 扫描并批量导入具体 `Host` 别名，不扫描 WSL 配置。
- 导入弹窗默认填充系统 `.ssh` 目录，也可选择其他目录；导入前可预览来源文件、跳过已存在别名、多选候选主机并选择目标 SSH 分组。完成后提示成功、失败和重复跳过数量；目录、文件或 Include 无效时显示本地化失败原因，不写入部分数据。
- 解析支持 UTF-8 BOM、CRLF/LF、多别名、递归 Include、相对/绝对路径、`~`、环境变量与通配符，检测循环引用并跳过通配符/否定 Host 及条件 Include。
- 默认目录继续由系统 OpenSSH 自动读取配置；自定义目录保存本机配置文件引用，连接测试、远程目录查询和 PTY/daemon 终端启动统一使用 `ssh -F`。配置路径不参与同步、备份导出或普通日志。

### 国际化 / 繁体中文支持

- 设置中的界面语言新增 `繁體中文`，持久化语言偏好与自动检测同时支持 `zh-CN`、`zh-TW`、`en-US`，并识别 `zh-HK`、`zh-MO`、`zh-Hant` 等繁体地区或脚本标记。
- 前端国际化以现有 `zh-CN` 文案为基底，通过 OpenCC 在运行时映射生成 `zh-TW`，避免重复维护两套中文词典；现有英文文案继续独立保留。
- 设置、统计看板、历史会话、Prompt Library、状态栏配置、桌面宠物联动、关闭按钮两类弹框及常见通知文案切换到繁体时不再误回退英文；桌宠窗口内的状态标签、操作按钮、宠物名称和目标标题也会随 `zh-TW` 转为繁体显示，并对少量简繁同形词做繁中术语覆写（例如状态“工作中”）。日期、时间和数字格式按当前语言 locale 输出，并继续强制 24 小时制时间显示。

## [V1.2.9] - 2026-07-18

### 桌面宠物

- 提供独立悬浮窗桌面宠物（窗口标签 `desktop-pet`），可在主应用外常驻显示；设置页可启用/关闭、选择宠物、调整尺寸（小/中/大）、置顶、锁定位置、显示状态文案与会话名、终端全屏时自动隐藏，以及重置/记忆窗口位置。
- 宠物情绪跟随终端 Tab 与 daemon 后台任务状态：执行中 → working、待处理 → waiting、完成 → success、失败 → error、空闲 → idle、无可用任务 → sleeping；同时汇总运行中/待处理数量，并提供任务列表点击跳回对应会话。
- 状态推导综合打开终端、持久化会话、后台任务与 PTY 输出活跃度；主窗口通过配置/快照事件与宠物窗同步，宠物窗可打开主窗口、打开设置或隐藏自身。
- 右键菜单采用紧凑纵向操作区和向左展开的毛玻璃列表；远程托管按“选择已启用平台 → 选择已停止的 Codex 会话”操作，支持 Telegram、飞书、微信与企业微信，并显示凭据、连接和机器人会话是否就绪。
- 支持 `.clipet` 宠物包（ZIP + `manifest.json` + 资源），引擎含 `image-v1` 与兼容 Codex Pets 的 `codex-sprite`；仅接受 png/webp/经清洗 svg，限制包大小与条目数，拒绝 HTML/JS/可执行文件、符号链接与路径穿越。
- 设置页图鉴支持远程目录、本地缓存与内嵌兜底，可按 SHA-256 / 大小 / 最低应用版本校验后下载安装；支持本地导入 `.clipet`、卸载 CLI-Manager 管理的宠物。内置 `builtin.cli-cat`，预置 terminal-robot / pixel-fox / mint-slime。
- 可发现本机 Codex Pets 目录中的外部宠物并加入已安装列表；外部 Codex 宠物只读展示，不支持应用内卸载。
- 工作状态上下抖动可单独开关，幅度 0–5 px（默认关闭）；关闭或设为 0 时保留宠物包自身动画，不叠加整体位移。

### 退出与后台任务

- “退出时也检测已完成的任务”默认关闭；开启后，退出检测会额外纳入 Claude/Codex Hook 明确标记为已完成或失败的会话，并允许保留到 daemon 后台任务中心。普通 Shell 命令结束、待处理通知和非 PTY 视图不会被误判为已完成 CLI 任务。

### Worktree 今日项目用量

- 终端实时统计中的今日项目用量按父项目路径与全部 active Worktree 路径一次聚合；主项目 Tab 和同项目 Worktree Tab 展示相同合计。后端按唯一历史会话统计，父目录包含 Worktree 时不会重复计算，当前 checkout 尚无最新历史会话时仍可展示项目级用量。

### 文件浏览器 Gitignore

- 文件浏览器优先读取项目根 `.gitignore`，支持标准的根路径锚点、任意层级目录规则、`*` / `**` 通配和 `!` 否定规则；Windows 本地与普通 UNC 项目按大小写不敏感匹配，WSL/POSIX 项目保持大小写敏感；忽略目录归入“已折叠文件”，忽略文件不在主文件树显示。
- 项目 watcher 报告 `.gitignore` 创建、修改或删除后会立即重载匹配规则；文件缺失或不可读时回退到内置缓存、构建产物和常见临时文件规则，手动忽略路径与默认折叠目录继续叠加生效。

## [TEMP] - 2026-07-13

### SSH 远端历史

- SSH Claude/Codex 项目可从统一历史工作区读取已绑定远端目录中的会话，支持分页列表、搜索、消息/工具调用/子 Agent 详情、Diff 与 usage 摘要；详情按需通过复用 Agent bridge 分块传输，列表摘要和统计事实进入现有 `history-catalog.db`，断线时明确显示 stale/offline 状态。
- 远端 Agent 只维护可重建的增量派生索引，使用 machine/user/source/config root 作为稳定来源身份，并通过跨进程单写锁、游标、轮转与 tombstone 避免重复全量扫描；不复制完整历史目录，不允许远端路径进入本地文件、Git、供应商、编辑或删除 API。
- 远端历史支持同源恢复：Agent 重新校验原 JSONL、远端 cwd、Host/用户/config root 与 Claude/Codex 可执行文件；匹配项目时创建新的交互式 SSH PTY，无匹配项目时可使用原远端目录。当前客户端已有同一会话会直接跳转，其他 consumer 正在运行时阻止并发恢复。

### 历史项目同步

- Codex / Claude 历史项目同步弹窗支持对已选择项目批量忽略并二次确认，也支持在单个项目行直接忽略；同步成功的项目会自动进入持久化忽略列表，后续同步扫描不再展示或提醒这些项目。

### 历史会话消息选择

- 会话消息批量删除模式支持点击整条可编辑消息进行选中或取消，保留选择框状态提示，并支持 Enter / Space 键盘操作；普通浏览模式和不可编辑的结构化消息不受影响。

### 历史会话恢复

- 恢复会话的项目配置选择弹窗支持按目录分组展开与折叠；输入搜索条件时自动展开匹配分组，关闭按钮使用轻量圆形图标样式。

### Hook 与 cc-switch 跨环境兼容

- Claude Code / Codex Hook 配置目录和 cc-switch 数据库可独立位于 Windows、WSL、macOS 或 Linux 的本机环境；Hook 命令按各 CLI 目标环境生成，数据库访问按数据库路径选择运行时。
- Windows 版可读取和更新 WSL 内的 `cc-switch.db`：读取通过对应发行版内 Python SQLite 在线备份后由 Rust 解析，Hook 与状态栏通用配置写入在 WSL 内使用事务和旧值校验完成，不通过 UNC/Plan 9 直接写 SQLite。
- WSL 中缺少 Python SQLite 运行时或发生并发更新时返回可诊断错误，不自动安装依赖、不回退到不安全写入。

### 历史用量与请求日志

- 历史用量分析使用应用窗口内全屏工作区，保留项目和时间筛选、日期下钻及既有统计能力；Token / 费用趋势独占中间整行，底部依次展示项目排行、模型排行、24 小时活跃和 Token 构成。
- 新增“请求日志”页签，将本机 Claude/Codex CLI 会话日志中的 usage 事件增量写入 CLI-Manager 自身 `request_logs` 表；应用启动同步一次，运行期间每 60 秒同步，并支持手动刷新。
- 请求日志支持来源、项目、模型、会话关键字和日期范围筛选，提供今天、近 7 天、近 30 天快捷日期和可折叠高级条件；紧凑工具栏、四项独立汇总卡片、吸顶表头、列内容居中、来源品牌图标、异常用量提示、来源/项目/模型快捷筛选，以及合并在固定操作列中的会话 ID 复制与历史会话跳转提升高密度日志检索效率。仅保存 usage 元数据，不保存 Prompt、响应正文、密钥、伪 HTTP 状态或伪耗时。
- 请求日志同步不调用、不读取、不依赖 cc-switch；费用按 CLI-Manager 当前模型价格动态计算，未匹配价格的 Token 单独标记为未定价。
- ccusage 看板改为应用窗口内全屏布局，仅重排现有趋势、峰值摘要、Token 构成、热点图和模型排行，不增加功能、指标或数据源。

### Claude 状态栏

- 设置页提供内置 Claude 状态栏可视化配置器，支持完整组件目录、最多三行布局、组件搜索、按当前选中行点击添加、从组件库拖到指定行或组件位置、行内删除、跨行拖拽和属性区跨行移动，以及颜色/粗体、自定义文本/符号/命令、Powerline 与实时预览。
- CLI-Manager 可执行文件通过隐藏的 `__statusline` 子命令直接接收 Claude Code 状态 JSON 并输出状态栏，无需额外安装 Node、Bun 或 ccstatusline-zh。
- 支持检测、安装和卸载 Claude `statusLine`，写入时保留其他 Claude 设置并创建备份；支持一次性导入 `~/.config/ccstatusline/settings.json` 的旧配置。
- 支持可视化编辑 Codex 原生 `/statusline` 对应的 `[tui].status_line`：选择模型、上下文、5 小时/周额度、Git、Token、权限、会话和任务进度等项目，调整显示顺序并预览后写回 `config.toml`；其他 Codex 配置保持不变。
- 状态栏设计器将 Claude Code 和 Codex 分成独立工作区：Claude 使用三行组件画布和属性编辑，Codex 使用原生项目选择与拖拽排序；共享实时预览支持 60–180 列宽度和全部内置终端主题，同时页面控件自动适配当前应用主题与配色。
- Claude Code 与 Codex 分别维护多份命名状态栏配置，支持保存即应用、复制、重命名、切换和删除非当前配置；首次进入自动采纳已有实际配置，检测到外部修改时可另存为新配置。
- 软件内文本输入与确认操作统一使用跟随应用主题的内部弹窗，覆盖状态栏配置、导入冲突、未保存内容切换、配置删除、字体安装和终端背景移除，不使用 WebView `window.prompt` 或 `window.confirm`。
- 支持将 Claude/Codex 状态栏配置库整体导出为版本化 JSON，并在导入时逐项选择覆盖、跳过或重命名；导入不会自动切换当前配置，也不会携带供应商密钥或其他 CLI 设置。
- Claude/Codex 实时预览使用本地化状态名称说明各项含义；Codex 预览位于原生状态栏标题下方。Claude Powerline 支持字体检测与用户确认安装、分隔符、起止端帽、组件对齐、跨行主题延续及 10 套内置主题，颜色选择显示色块和中英双语名称。
- Claude Code 实际状态栏为数据组件显示中文短标签，Git 分支使用 `⎇` 图标；上下文组件以 16 格进度条展示已用量、上下文上限和使用百分比；Powerline 首组件左侧保留三个空格，其他间距保持紧凑。
- Hook 与状态栏会先写入本地 CLI 配置，再自动尝试写入 cc-switch 通用配置；通用配置指 cc-switch 在切换供应商时合并到所有启用“应用通用配置”的供应商里的共享片段。未配置或未检测到 cc-switch 时，本地配置文件写入仍作为托底生效。
- 安装、卸载、保存或切换 Claude 状态栏时会同步 CC Switch `common_config_claude`，切换供应商后仍保留 CLI-Manager 状态栏；同步仅处理带 `__statusline` 标记的配置，不覆盖其他通用字段、Hook 或第三方状态栏。Codex 原生状态栏会同步到 `common_config_codex` 的 `[tui].status_line`，并保留现有 `[features]`、Hook 信任块和其他 TOML 表。

### 终端文件路径快捷打开

- 终端输出中的 Windows、UNC/WSL UNC 及可转换的 Linux 绝对文件路径支持 `Ctrl + 单击`；路径尾部的 `:行号`、`:行号:列号` 及其后的源码片段会被自动剥离；悬停时按真实目标类型显示文件或文件夹图标，打开后统一使用 Windows 资源管理器定位目标。

### 终端渲染稳定性

- 终端采用 VS Code 风格分层：React 终端宿主统一通过 `TerminalProcessManager` 访问 PtyHost；WebView 使用鉴权二进制 WebSocket 接收输出与 Replay，高频输出不经过 Tauri Base64 事件。
- daemon 对输出进行 5ms 合并并使用序列号、字符 ACK 与 100000/5000 高低水位背压；attach Replay 记录输出发生时的列数、行数和序列，按 Replay 后实时输出的顺序恢复。
- Attach 使用 replay/attached/live 写入屏障；XTerm 尚未提交的帧由 `TerminalProcessManager` 跨组件重挂持有，只有 xterm write callback 完成后才按序 ACK。初始与断线重连 Replay 都按历史 resize 串行恢复，完成当前容器 fit 后才继续 live 输出，历史尺寸不会写回 live PTY。
- daemon 每客户端使用独立 writer queue，慢客户端不会持全局锁阻塞其他会话；内存 Replay 超过 2 MiB 后将更早的完整安全帧写入会话 spool，关闭会话或启动新 daemon 时清理，不静默裁剪活跃会话输出。
- WebSocket 具备鉴权、请求、心跳超时与自动重连；关闭请求先建立 tombstone，失败或断线后不会把已关闭 Tab 重新 attach，最后会话关闭时同步取消待执行重连，避免残留空闲连接与心跳。UI 会话对账不终止 daemon 后台任务。
- 隐藏终端持续交给 xterm 解析输出，不再保存隐藏输出后批量重放；低内存模式只延迟释放 WebGL，切回时恢复渲染与视口。大缓冲区纵向 resize 立即生效，横向 reflow 采用最长 34ms 的首帧 + 尾帧实时节流；macOS/Linux 同步重排光标所在逻辑行。快速缩小时会在每次实际 resize 前保留最近一张正确画布并随容器实时拉伸，PTY 输出提交后更新该画面，尺寸稳定 72ms 后再显示底层渲染器，避免暴露 xterm/WebGL 把左侧内容短暂画到右侧的中间帧。
- 横向 resize 前会为用户当前查看的 normal-buffer 首行建立临时 marker，等待 xterm 完成异步视口同步后恢复到同一逻辑文本位置；实时底部跟随和 alternate buffer 不受影响。
- 拖动顶层 Workspan Tab 悬停其他 Workspan 500ms 后会自动激活目标，可继续拖到目标 Pane 边缘完成左、右、上、下分屏；Pane 内 Terminal Tab 继续使用相同的中心移动与边缘分屏落点。
- 普通 Tab、Workspan 或隐藏终端切回时先等待 xterm 自然恢复完整 viewport；两帧内没有完整渲染才触发强制刷新，并在恢复期间保持绘制层遮罩，避免大片空白或每次切换都逐行重绘。
- 左侧边栏拖动期间直接预览 DOM 宽度，松手后才提交 React 状态和持久化；终端 Pane 不再对 left、top、width、height 使用几何动画，降低连续 ResizeObserver 与 xterm reflow 开销。
- Windows PTY 直接调用 ConPTY API，并继续支持兼容 ConPTY DLL；macOS/Linux 使用 `openpty`、控制终端和进程组。生产路径不再依赖 `portable-pty`。

### SSH 远程项目与主机管理

- 设置页新增 SSH 主机管理，支持地址连接和 SSH Config 两种来源；认证支持 SSH Agent、私钥、用户/密码、每次询问密码和 Keyboard-interactive/MFA。用户/密码默认保存到系统凭据库，系统 OpenSSH 通过一次性 AskPass 通道读取，密码不进入 SQLite、日志、同步数据或命令行。
- SSH 主机支持多级分组树展示，可新增根分组和子分组；新增/编辑主机弹框固定高度，顶部标签随滚动联动，连接来源切换不丢失已输入字段，测试结果显示在测试按钮旁并按成功/失败/等待使用不同颜色。
- “新增终端”支持本地项目 / SSH 远程项目切换；SSH 远程项目可选择任意支持的 SSH 主机认证方式，包括用户/密码，并支持远程路径检测和目录浏览。项目列表保持原有分组结构，不额外显示 SSH 徽章。
- SSH 远程终端由 Rust 结构化 Launch Plan 生成系统 OpenSSH 参数，可在远程目录启动 CLI、使用分屏与 Workspan，并由 daemon 托管与恢复。远端 Claude/Codex Hook 已通过显式安装的 `cli-manager-ssh-agent` 接入；SSH 文件面板支持 Agent 约束下的只读树、文件名/代码搜索、文本/图片预览和远端路径复制，Git 面板支持只读仓库、状态、Diff、分支和 upstream/asOf，终端实时统计与今日项目用量复用远端历史 Agent/catalog；Worktree 和远程资源监控仍保持禁用或降级提示。

## [V1.2.7] - 2026-07-11

### Linux 图形兼容与分发

- Linux 启动会在 WebKitGTK 创建前解析图形策略；自动模式仅在 Wayland + NVIDIA 专有驱动环境设置 `__NV_DISABLE_EXPLICIT_SYNC=1`，并支持系统默认、禁用 DMABUF 和禁用加速合成三级手动降级。用户已有的标准 WebKit/NVIDIA 环境变量优先，不会被应用覆盖。
- 开发者设置在 Linux 下提供图形兼容模式和可复制诊断，显示会话类型、桌面环境、NVIDIA 判定、请求模式、实际模式和已应用 workaround；修改模式后重启生效。
- Linux 主窗口默认在首屏就绪后显示；启动超过 3 秒会显示加载阶段，单阶段超过 15 秒展示可重试错误，不再以白屏或黑屏代替启动反馈。
- 终端 WebGL context loss 后回退到默认渲染器；NVIDIA Wayland 或显式 WebKit 降级模式下，隐藏终端延迟释放 WebGL 资源但保留 PTY、scrollback 和后台输出缓冲。
- 仓库提供 `cli-manager-bin` AUR 包装模板。AUR wrapper 会把安装标记为包管理器渠道，禁用应用内下载安装，升级由 AUR helper 或 pacman 负责。

### PTY 守护进程与后台任务

- 应用启动后会连接或拉起 `cli-manager-daemon` 独立守护进程；内置终端 PTY 可由 daemon 托管，应用真退出后任务继续运行，重开应用后优先 attach 原会话并按“已有回放 -> 恢复期间实时输出”的顺序恢复画面，不重跑启动命令、不重复 resume。
- daemon 与安装包一起作为 sidecar 分发，Windows、macOS、Linux 用户不需要单独下载；开发版和安装版使用独立发现文件，互不 attach。
- Windows 下 daemon 使用 Job Object 兜底治理子进程树；daemon 正常退出或被强杀时，系统会回收其托管的 PTY 子进程，降低孤儿进程风险。
- 退出应用时如存在运行中任务，会在一个弹窗中提供“退出应用，任务在后台继续执行”“最小化到托盘”“终止任务，退出应用”：后台继续会真退出 UI 并保留 daemon 任务，最小化到托盘不关闭任何终端，终止任务会关闭 PTY 并清理恢复快照但不删除 Claude/Codex 原始历史。
- 终端右侧工具栏提供后台任务中心；入口可通过“设置 -> 侧边栏 -> 终端工具栏”控制显示和排序，默认在无后台任务时禁用、有任务时显示呼吸提示。
- 后台任务中心展示任务名称、项目、创建时间和任务状态，支持恢复、丢弃或删除；恢复后的 Claude/Codex Tab 保留对应 CLI 图标，任务完成后仍保留在列表中，只有用户手动恢复、删除或丢弃后才消失。
- 后台任务状态由 daemon 根据 hook 上报维护，区分执行中、待处理、已完成和失败；PTY/shell 是否仍存活只作为兜底，不再把 shell 存活误判为任务仍在执行。
- Claude/Codex hook 会上报到 daemon 的稳定 hook 端口；应用未运行时 daemon 缓存最近 hook，PermissionRequest 可唤起应用并定位到对应后台会话。

## [V1.2.8] - 2026-07-09

### 历史会话消息编辑

- 历史会话详情按钮与列表右键菜单恢复会话时，会匹配来源和目录对应的项目配置并携带其 CLI 启动参数、供应商覆盖、环境变量和 Shell；多个匹配配置通过支持搜索与分组的项目选择弹窗确认后再启动；没有匹配项目时展示全部项目和“使用新窗口”，使用新窗口会先进入历史会话工作目录再执行恢复命令。
- 历史会话详情的聊天气泡支持消息级操作：悬停显示复制、编辑、删除和"在此后插入"操作栏；工具调用、思考等结构化消息只提供复制，不开放编辑入口，避免破坏 tool_use/tool_result 协议配对。
- 编辑在气泡内以行内文本框完成（Ctrl+Enter 保存、Esc 取消），保存后直接写回 Claude / Codex 本地会话 JSONL 原始行：仅替换文本块内容，uuid、parentUuid、usage、图片等其他字段与非文本块原位保留；`claude --resume` / `codex resume` 后 AI 基于修正后的历史继续。
- 删除消息会自动修复 Claude parentUuid 会话链（后继消息重链到被删消息的父节点）；Codex 会同步删除配对的 `event_msg` TUI 重放行。
- 支持批量删除：详情页顶部进入选择模式后勾选多条消息一次性删除；批量删除为单次原子写回，任一目标行校验失败整批拒绝，连续删除时 Claude 会话链跨代上溯重链到幸存祖先，Codex 同步移除全部配对行。
- 插入消息可选用户 / 助手角色，在锚点消息之后注入上下文：Claude 侧生成新 uuid 并接管锚点子链，Codex 侧同步写入 `response_item` + `event_msg` 双行，保证模型上下文与 TUI 重放一致。
- 首次修改某个会话文件前自动备份原文件到 `.cli-manager/history-backups/`；详情页"编辑记录"弹层以字符级 diff 高亮（删除红色删除线、新增绿色底色，−/＋ 行符）展示每次操作差异，支持单条撤回（编辑恢复原文本 / 插入移除消息 / 删除就近恢复消息），也支持一键还原到编辑前的原始文件。
- 双守卫防写坏：写入前校验文件指纹（外部改动过则拒绝并自动重载会话）与目标行角色 + 规范文本（行号漂移即拒绝）；写回使用临时文件 + 原子替换。
- 正在终端中运行（hook 绑定 sessionId）的活跃会话，编辑 / 删除 / 插入前会弹出警告确认；收藏快照兜底的只读会话不提供编辑入口，收藏会话编辑后快照自动同步。

- 首次启动且尚未维护项目时，应用会延迟扫描本机 Claude / Codex 历史项目，并弹出同步选择窗口供用户确认要纳入管理的项目；已维护项目列表时启动不再自动扫描或提示。
- 同步记录会持久化保存；后续在历史会话列表点击刷新时会同时检测缺失的历史项目，也可移除已同步历史记录。
- 左侧项目列表支持展示同步来的历史分组；这些分组可打开聚合同项目历史上下文的只读终端。

### 用量分析与模型价格

- 历史统计、实时统计和今日项目用量按本地模型价格表估算费用；裸 GPT 版本模型带 reasoning effort 时会按 `模型名(effort)` 独立价格处理，例如 `gpt-5.4(high)`、`gpt-5.6(xhigh)`；`gpt-5.3-codex-spark` 这类带用途后缀的模型保留原模型名。未配置对应价格时计入未定价 Token。
- 实时统计和历史详情侧栏的“今日项目用量”会展示总 Token、费用，以及输入、输出、缓存命中、缓存写入和未计价 Token 明细，便于判断费用偏低是否由缓存命中或未计价 Token 导致。
- 模型价格页支持识别本地模型后只同步缺失价格，或按当前筛选/搜索范围同步；远程基础模型价格可作为 effort 变体候选手动确认，不会自动套用。

### Hook 通知

- 系统级 Hook 通知使用短正文并截断过长详情，避免 Windows 通知内容不需要滚动时仍出现滚动条。
- Hook 设置支持“窗口聚焦时不发送系统通知”开关；开启后 CLI-Manager 主窗口已聚焦时只显示应用内通知，关闭后即使窗口聚焦也会按系统通知设置发送操作系统级通知。
- 应用内 Hook toast 会取消外层通知容器的滚动裁剪，保持卡片自身控制文本省略和圆角裁剪。
- Hook 设置页支持“三方 Hook 通知”：可配置多组 DingTalk、Feishu、WeCom、Bark、PushPlus、WxPusher、ServerChan、Telegram、ntfy、Gotify 和 Custom HTTP 目标，按事件筛选并测试发送。
- 三方 Hook 通知支持全局启停开关；关闭后保留所有目标配置和凭证，不自动派发，方便随时恢复。
- 第三方通知由实际接收 Hook 的主进程或 daemon 异步派发；daemon 托管后台任务时，即使前端未连接也能继续发送命中的第三方通知。
- 第三方通知只发送带 emoji 的安全摘要（CLI 来源、项目名、事件、时间、通知 ID），不发送 Prompt、终端输出、绝对路径、session/tab id 或 transcript；失败不会影响本地 Hook 状态、toast 或系统通知。
- 三方 Hook 通知的全局开关、目标、事件筛选和完整平台凭证可通过 WebDAV 或本地 zip 同步；下载时可独立选择是否恢复该域，旧快照缺少对应字段时保留本地配置，远端目标恢复前会过滤未知平台、重复 ID、畸形记录和超过 20 个的记录。

- 历史会话原文中的本地图片标记会优先以内联缩略图展示，点击可放大预览，预览中按 Esc 会优先关闭图片，并保留图片标记后的正文；图片缺失或读取失败时降级显示原始标记文本。
- 历史会话详情支持在同一行编辑会话别名、标签并保存；标签输入可选择已有标签，也可继续手动输入新标签。
- 历史会话全局搜索支持直接按 `sessionId` 定位 Claude / Codex 历史会话。
- 历史会话详情“变更”页读取当前会话 JSONL 中的结构化文件操作，支持还原 Codex 工具调用中的转义 Patch，并按操作时间列出真实文件路径和增删行数；文件图标与文件浏览器一致，彩色 Tag 区分新增、修改、删除，增加行与删除行分别使用绿色和红色；右键可跳转到产生修改的原文对话；点击文件时先将 Apply Patch 转为标准 unified diff，再使用文件浏览器同款只读 Diff 查看器进行左右分栏对比。
- WSL 中的 Codex 历史会话支持查看、恢复后的实时回读和 Claude -> Codex 转换；应用以 rollout `cwd` 识别真实项目，转换索引使用 Linux 路径，并避免从 Windows 跨 UNC 写入 Codex WAL 状态库。

### Git 变更面板

- Git 变更面板当前分支区域支持打开分支菜单，查看本地分支和远程分支。
- 分支菜单提供搜索框，可按本地分支名、远程分支名或 upstream 快速过滤；大量分支时先渲染前 80 条并提示继续搜索，减少菜单卡顿。
- 分支菜单支持切换本地分支；从远程分支 checkout 时会创建本地跟踪分支。
- 分支菜单支持基于搜索框中的新分支名显式新建并切换分支。
- 分支菜单支持执行 `fetch --prune` 刷新远端分支和提交信息，不会合并远端提交或修改工作区。
- 切换分支遇到本地改动会被覆盖时不会强制 checkout；Git 面板支持 Smart Checkout 确认流程，可在用户确认后先 `stash push -u`，切换分支，再 `stash apply` 恢复改动，并保留 stash 记录作为兜底恢复点。
- Smart Checkout 的 stash/apply 阶段如果失败或产生冲突，会展示明确错误并刷新 Git 面板状态，便于继续处理冲突；stash apply 冲突不会再误用 pull/merge 的中止操作。
- Git 变更文件列表的文件右键菜单支持打开源文件，会直接在文件编辑器 Pane 中打开对应项目文件；删除状态文件不会触发打开。
- 未跟踪文件和未跟踪目录的右键菜单支持从磁盘删除；删除前会二次确认，后端只允许删除当前 Git 状态仍为未跟踪的仓库内文件。
- 当前活动页是项目文件编辑器时，右侧 Git 变更面板会关联到该文件编辑器所属项目。
- WSL UNC 项目如果实际软连接到 `/mnt/<盘符>/...` 的 Windows 工作区，Git 变更面板会回退到 Windows/native Git 状态，避免 WSL Git 将全部文件误判为修改。

### Git Worktree 并行隔离

- 项目支持 Worktree 隔离策略：不提醒（默认）、提醒、并行时自动隔离、始终自动隔离。
- 项目已配置任意 CLI 工具（不限 Claude/Codex）且已有同项目终端时，新开终端可提示或自动创建独立 worktree，避免多个 AI 任务修改同一工作区；选择“不提醒”时始终直接普通打开；普通未配置 CLI 的终端项目不会因启动命令或运行状态触发隔离。
- 设置 -> 开发者关闭“Worktree开发”后，全局忽略项目隔离策略并直接普通打开，但不删除已有项目配置、Worktree 记录或手动 Worktree 操作入口。
- Worktree 使用项目同级 `<项目名>-worktrees/<任务名>` 目录和 `wt/<任务名>` 分支，也支持项目级自定义根目录。
- 项目树显示 worktree 子条目；项目级供应商徽章显示在主项目行，Worktree 级供应商徽章只显示在对应 worktree 子项；终端 Tab 仅使用 Worktree 任务名作为标题，并显示 worktree 徽标。
- Worktree 右键菜单支持打开目录、浏览文件、切换供应商、会话历史、安装依赖、完成任务和丢弃；切换供应商只影响当前 worktree，浏览文件会进入 worktree 独立目录，查看历史时项目筛选仍显示主仓库项目，实际列表限定在 worktree 独立目录，历史列表会用 `WT` 标记 worktree 会话。
- Worktree 终端 Tab 下，右侧实时统计、Git 分支查询和文件面板使用 worktree 自己的目录；“今日项目用量”按 worktree 实际路径聚合；已绑定 CLI sessionId 的实时统计严格匹配当前 session，不回退到同项目其他会话。
- 完成任务向导支持提交全部改动、合并回主工作区和清理 worktree/分支；主工作区脏时阻止合并并给出可读处理建议，冲突时自动中止合并、列出冲突文件并说明主工作区已恢复到合并前状态。
- 完成任务时，如果 Worktree 与主工作区没有内容差异，会阻止无意义合并并提示无需合并；Worktree 没有未提交改动时可直接进入合并/清理阶段，不再要求输入提交信息。

### Git Worktree 任务管理

- 项目 Worktree 设置可勾选“开启 Worktree 后检测依赖安装”；默认不勾选时，新建或打开 worktree 不会自动检测或弹出依赖安装提示。
- 对 worktree 右键选择“安装依赖”时，会在该 worktree 路径打开专用终端 Tab 执行检测到的安装命令；若无需安装会直接提示。
- 包含 worktree 的项目行支持展开/收起子列表，折叠状态会随侧边栏项目树状态保存。
- Worktree 使用线性分支图标展示，项目树、折叠侧栏浮层、拖拽预览和终端 Tab 徽标保持同一图标语言。
- 终端 Tab 中的 Worktree 徽章操作菜单会跟随终端主题色显示，与 Tab 右键菜单保持一致。
- Worktree 终端 Tab 与项目树联动：切换 Tab 时会自动选中左侧对应 worktree，点击已打开的 worktree 子项会切回对应 Tab。
- Worktree 历史入口不改变普通会话恢复/继续对话逻辑；普通项目会话仍按原项目目录打开历史和恢复上下文。
- 丢弃 worktree 前会关闭关联终端页签；后端会对 Windows 文件锁类删除失败做有限重试，包括 `os error 32` 和 stale 目录直接删除；若 Git 登记已清除但只残留空目录，或 Git 仍登记 path/branch 但目录 `.git` 已损坏，会作为 stale worktree 清理并删除对应 `wt/` 分支。

### 文件浏览器搜索与菜单

- 文件浏览器搜索框支持在输入框内直接切换“文件 / 代码”搜索模式。
- 项目文件预览、编辑、代码搜索和 Git Diff 支持自动识别 UTF-8/BOM、UTF-16 BOM 以及 GBK、Big5、Shift_JIS、Windows-125x 等常见文本编码；编辑保存保持原编码和 BOM，无法映射的新字符会被阻止写入。非 UTF-8 Diff 仅关闭可能破坏原始字节的行级/Hunk 级回滚，整文件操作仍可用。
- 文件浏览器搜索框与右键菜单会跟随所在侧边面板主题配色，避免深色侧边栏中出现浅色控件割裂。
- 文件路径、Git 状态和顶部操作按钮的悬浮提示使用应用主题背景、边框与文字颜色，浅色与深色主题下保持一致可读性。
- 文件浏览器会将已加载层级中的常见缓存、构建产物、AI 工具目录及手动忽略目录统一归入文件树底部的单一“已折叠文件”聚合行，并用相对路径区分不同父目录下的同名项。
- 文件树、文件搜索结果、代码搜索结果和根空白区域右键菜单支持打开所在文件夹；文件项会在系统文件管理器中选中目标文件。
- 文件浏览器顶部项目区域支持双击打开项目位置。
- 从文件浏览器拖拽文件/目录到内置终端时，拖拽预览按动画帧紧跟指针，并插入与右键“复制 AI 路径”一致的路径格式（不带项目名的 `@相对路径`，目录带尾 `/`）。

### 0.1 终端外部文件拖拽

- 从系统文件管理器拖拽文件到内置终端时，会在终端输入位置显示本地文件路径。
- 多文件拖入时会按空格拼接多个路径。
- 路径会按当前终端 Shell 使用对应引号规则，降低空格路径或特殊字符路径输入失败的概率。

### 0.2 文件浏览器自动刷新

- 打开项目文件浏览器后，终端或外部工具修改并保存项目文件时，文件树会自动刷新，不再依赖手动点击顶部刷新按钮。
- 已展开目录会尽量保持展开状态，不因自动刷新而整棵树塌陷。
- 已打开且没有未保存修改的文件预览会自动重读最新磁盘内容。
- 已打开但仍有未保存修改的文件不会被自动覆盖。
- 文件 watcher 会携带变更路径，文件浏览器优先刷新受影响目录。
- 连续文件系统事件会串行合并后刷新，避免一次保存或批量生成文件触发多次重载和刷新乱序。
- `node_modules`、`target`、`dist`、`.gitnexus` 等高噪音目录变化不会触发文件树刷新。
- 当本机 watcher 不可用时，会自动降级为低频轮询，并在窗口重新聚焦时补一次刷新。

### 0.3 文件浏览器 Git 状态

- 文件树和文件搜索结果会标记 Git 状态，覆盖修改、新增、删除、重命名、冲突和未跟踪文件。
- 文件状态使用不同颜色区分：新增为绿色、修改为蓝色、删除为红色，未跟踪和冲突使用独立颜色。
- 目录会聚合其子文件的 Git 状态，折叠时也能看出目录内存在变更。
- 打开有 Git 变更的文本文件时，编辑器左侧标记和概览标尺会显示新增、修改、删除位置。
- 有 Git 变更的文件可通过右键菜单在文件编辑器 Tab 中打开 Diff。
- 内嵌 Diff 支持文件级、代码块级和选中行级回滚，并跟随当前终端主题背景。

### 0.4 Git 面板多子仓库监控

- 项目根目录下存在多个 git 仓库时，Git 面板顶部显示仓库切换下拉，列出主仓库与限深扫描到的子仓库（含当前分支）。
- 切换子仓库后，变更列表、分支状态、暂存、提交、推送、拉取与 Diff 均针对所选子仓库生效。
- 扫描排除 `node_modules`、`target`、`dist` 等常见大目录并跳过符号链接，避免面板首开卡顿。
- 子仓库内文件改动会触发面板自动刷新；子仓库 `.git/objects` 等内部写入不会引发刷新风暴。
- 单仓库项目 UI 保持不变；主仓库变更列表不再展示嵌套子仓库目录条目。
- 文件 Diff 弹窗支持按 Esc 键关闭。

- SSH Config 连接由系统配置完整托管；手动连接支持 Agent、指定私钥、密码和 Keyboard-interactive，认证方式切换会清理失效参数，主机列表可直接打开 SSH 终端完成密码、MFA 或私钥口令交互。
- SSH 登录密码可保存到系统原生凭据库：Windows Credential Manager、macOS Keychain、Linux Secret Service；OpenSSH 通过一次性 AskPass 通道读取，数据库和同步数据只保留凭据引用。
- SSH 主机支持系统 OpenSSH 探测和分阶段连接诊断；密码、私钥口令和 Token 不写入 SQLite、日志或会话快照。
- 新增终端弹窗通过“本地项目 / SSH 远程项目”Tab 切换；两类表单分别保留草稿，编辑已有项目时锁定运行环境类型。
- SSH 远程项目选择已有主机和绝对 POSIX 远程目录，支持连接后浏览、手工输入、目录存在/可进入/Git 仓库检测；密码或 MFA 模式无法执行非交互浏览时降级为手工路径。
- 远程终端由 Rust 根据结构化 Launch Plan 生成 OpenSSH 参数，安全进入远程目录并启动项目 CLI；前端不拼接完整 SSH 命令。
- SSH 会话支持普通 Tab、复制 Tab、终端分屏、Workspan、项目分组、命令模板和 daemon 后台托管；Tab hover 展示 SSH 主机、远程目录、连接状态和断开原因。
- SSH 会话状态包括连接中、等待认证、已连接、已断开和连接失败；区分远端 Shell 退出、远端命令异常、SSH/网络中断和本机 SSH 进程异常。
- daemon 仍持有的 SSH PTY 在应用重开后直接 attach；已退出会话只显示断开状态并回放输出，不重新执行启动命令。
- 每台 SSH 主机可分别配置 Claude 与 Codex 的“CLI 配置目录（Hook 与历史）”，默认使用远端用户的 `$HOME/.claude` / `$HOME/.codex`；SSH 项目可设置独立覆盖目录。启动时按项目覆盖、主机配置、CLI 原生默认的顺序解析，并安全注入 `CLAUDE_CONFIG_DIR` / `CODEX_HOME`；保存或浏览目录不会安装 Agent、写入 Hook 或扫描供应商。
- SSH 主机“CLI 集成”页提供显式 `cli-manager-ssh-agent` 检测；打开页面不会自动建立 SSH 连接，点击检测后通过复用认证/跳板机/代理/AskPass 的一次性 `ssh -T` 读取 `version/status/doctor`。检测限制 8KB 登录提示、64KB stdout 和 8KB stderr，校验 Agent 身份、协议主版本及 Linux x64/arm64 目标，只持久化脱敏状态与版本元数据。
- `cli-manager-ssh-agent` 支持按 Host 显式预览并执行安装、升级、回滚和卸载；可使用默认 XDG 目录或自定义绝对/`~/...` 安装根。桌面端先验证 Minisign manifest、协议、目标、制品大小与 SHA-256，再通过当前 SSH 配置的 stdin 上传；远端使用安装锁、discovery record、版本目录和 `current/previous` 原子链接，默认禁止降级并在切换失败时恢复旧状态。发布页同时提供可先下载审阅的 POSIX 安装脚本，支持 HTTPS、显式受信 HTTP 镜像、dry-run、自定义目录和卸载；Agent 生命周期操作本身不会修改 Claude/Codex Hook。
- SSH 主机“CLI 集成”可按 Claude/Codex 及项目覆盖目录分别检查、预览、安装/升级和卸载远端 Hook，展示规范化配置根与实际 `settings.json`、`hooks.json`、`config.toml` 地址和指纹；只精确管理包含当前 Agent installation id 的条目，保留第三方字段、顺序、matcher、symlink 目标和用户自有 Codex feature。显式安装可创建缺失的标准默认目录，自定义缺失目录拒绝；修改根目录后旧 Hook 进入显式待清理状态，Agent 卸载会阻止遗留自有 Hook。
- SSH Claude/Codex 启动由 Rust 覆盖注入 Host/client/project/Tab/bridge epoch 绑定；普通 SSH、IDE、tmux 或另一个桌面客户端缺少匹配绑定时 Hook 快速 no-op，不写 spool。只有有效配置根已验证为 Hook `installed` 且 Agent 身份仍匹配时才建立每主机一个 protocol 1.1 Agent bridge；全局最多 4 条 bridge、2 条并发连接/重连，具备有界 preamble/hello/响应超时、10 秒 heartbeat、抖动退避、旧 socket 接管和最后会话回收。断线事件按 Host/client/installation 隔离保存，采用 24 小时、10000 条、32 MiB 上限、逐条有界读取、流式 ACK、sequence/event id、去重和 gap 告警；远端 transcript 引用保持不透明，不进入本地文件 API 或第三方通知。
- SSH 主机删除会解除项目的本机 Host 绑定并保留远程目录；活动终端或跳板机仍引用时禁止删除，远端 Agent/Hook 不会被隐式卸载，远端集成记录保留为待重新绑定状态。
- SSH 文件和 Git 面板均为只读能力：远端 Agent 只接受受约束的 root/relative path，跳过 symlink，限制目录条目、搜索结果、遍历文件数、单文件大小和 Diff 大小；保存、创建、重命名、删除、粘贴/移动、拖拽、系统 Explorer/Finder、Git mutation/network/credentials、external diff/textconv 与 Worktree 操作在 UI、store 和 Agent 层均拒绝。远端历史详情与统计在 Agent 在线时增量同步，断线保留 catalog 摘要并显示 stale/offline；外部 Windows Terminal 和远程资源监控仍保持禁用或降级提示，远程路径不会传给本地文件/Git/历史 API。
- 系统资源面板在 SSH 项目中仍可查看，但明确标注为“本机资源”，不代表远端服务器资源。
- WebDAV/本地导入导出保留 SSH 项目的远程目录与项目配置，但不同步 SSH 主机、凭据引用和私钥路径；导入到其他设备后项目显示重新绑定提示。

## 6. Claude Code / Codex CLI 集成

### 6.1 Hook Bridge

- 应用启动本地回环 Hook 通知服务
- 仅监听 `127.0.0.1`
- 使用 bearer token 校验 Hook 请求
- 为每个 PTY 会话注入：
  - `CLI_MANAGER_TAB_ID`
  - `CLI_MANAGER_NOTIFY_PORT`
  - `CLI_MANAGER_NOTIFY_TOKEN`

### 6.2 Claude Code Hook

支持事件：

- `UserPromptSubmit`
- `Notification`（细分 matcher：permission_prompt / idle_prompt 置 attention，auth_success 等不干扰）
- `Stop`
- `StopFailure`
- `SessionStart`（会话启动 / 恢复时回传 sessionId 并绑定到终端 Tab）

能力：

- Claude 提交用户 Prompt 时标记对应 Tab 为运行中
- Claude 需要处理时标记对应 Tab 为待审批
- Claude 完成时标记对应 Tab 为已完成
- Claude 失败时标记对应 Tab 为异常退出
- 支持悬浮通知卡片
- 切换到目标 Tab 后自动清理通知
- Hook running 状态增加 30 分钟超时回退
- Hook 事件按 timestamp 丢弃乱序旧事件
- Hook 上报脚本优先使用 `curl.exe`（失败回退 `Invoke-RestMethod`）
- Hook 命令加 `-NoProfile` 降低延迟

### 6.3 Codex CLI Hook

支持事件：

- `UserPromptSubmit`
- `PermissionRequest`
- `Stop`

能力：

- Codex 提交用户 Prompt 时标记对应 Tab 为运行中
- Codex 权限请求时标记对应 Tab 为待审批
- Codex 停止时标记对应 Tab 为已完成
- 与 Claude 共用终端 Tab 通知逻辑
- 按来源区分 Claude / Codex 通知

### 6.4 Hook 设置

- Claude / Codex Hook 设置拆分展示
- 支持选择 Claude 配置目录
- 支持选择 Codex 配置目录
- 支持安装 Claude Hook
- 支持卸载 Claude Hook
- 支持安装 Codex Hook
- 支持卸载 Codex Hook
- 展示 Hook 安装状态
- 展示「会话启动 Hook（SessionStart）」安装状态检查项
- 展示脚本路径
- 展示配置文件路径
- Claude Hook 安装时合并写入 `settings.json`，不删除用户自定义 Hook
- 安装 Hook 时自动清理旧版本注册条目
- Codex Hook 安装时写入 `hooks.json`
- Codex Hook 安装时确保 `config.toml` 中 `[features].hooks = true`
- Hook 会自动写入本地 CLI 配置并同步 cc-switch 通用配置；Claude Hook 同步到 `common_config_claude.hooks`，Codex Hook 同步到 `common_config_codex` 的 `[features].hooks` 与 CLI-Manager 拥有的 `[hooks.state.*]`，用于在 cc-switch 切换供应商后保留 Hook。未配置或未检测到 cc-switch 时，本地 Hook 配置文件写入仍作为托底生效。
- Codex 未识别 / 未选择有效配置目录时禁用安装按钮，不自动创建默认 `~/.codex`
- Hook 弹框全局开关
- Hook 弹框自动关闭时间配置

### 6.5 Hook 环境注入

- 为每个 PTY 会话注入 Hook 环境变量（按需）
- `pty_create` 新增 `hook_env_enabled` 参数
- 仅当对应工具 Hook 状态为 `installed` 时才注入 `CLI_MANAGER_NOTIFY_*` 等环境变量
- 空终端和未安装 Hook 的项目按普通 shell 处理

- 旧 `autoSyncOnClose=upload` 自动迁移为退出自动备份，其余旧自动同步动作关闭

### 13.6 本地 ZIP 与明文策略

- 快照不做应用层加密；项目环境变量、命令模板、命令提示 API Key、三方 Hook Webhook/Token/Secret 会以明文保存

- 中英文界面均明确提示使用 HTTPS WebDAV 并妥善保管 ZIP

## 20. 安全与边界处理

### 20.1 历史会话安全

- 删除历史会话前校验来源
- 删除历史会话前校验项目边界
- 删除历史会话前 canonicalize 路径
- 只允许删除受支持来源下的 `.jsonl` 文件
- 切换 Claude / Codex 配置目录后隔离历史索引与缓存

### 20.2 Hook Bridge 安全

- Hook 服务仅监听本地回环地址
- Hook 请求必须携带 bearer token
- Hook body 有大小限制
- Hook header 有大小限制
- 只接受指定事件类型

### 20.3 背景图片安全

- 图片来源必须是绝对路径
- 只允许 `jpg` / `jpeg` / `png` / `gif`
- 禁止 `webp`
- 保存后的相对路径禁止：
  - 空路径
  - `..`
  - 反斜杠
  - 绝对路径
  - 逃逸 `backgrounds/` 目录
- asset protocol scope 锁定到背景图目录

### 20.4 WebDAV 安全与稳定性

- 使用 Basic Auth
- 密码通过 keyring 存 Windows 凭据管理器、macOS 登录钥匙串或 Linux freedesktop Secret Service；原生 WSL 使用专用 collection 规避 default collection 缺失，不进 tauri store / SQLite / 同步快照
- 项目环境变量、命令模板、命令提示 API Key 与三方 Hook 通知凭证会按产品策略明文进入 WebDAV V3 快照与本地 ZIP 的 `snapshot.json`，不做应用层加密；HTTP 无传输加密，HTTPS 仅保护传输链路
- WebDAV 响应大小限制
- 请求超时限制
- 上传前确保远端目录存在
- 远程下载/删除仅接受备份目录下严格解析的快照文件名；旧 V1/V2 数据只允许显式导入

### 20.5 PTY 稳定性

- PTY 输出避免切断 UTF-8 字符
- PTY 输出避免切断 ANSI / OSC / CSI 转义序列
- 未完成残尾延迟到下一轮拼接
- 异常源端有缓冲兜底，避免无限增长
- resize 有最小列数 / 行数保护

## 23. 简短结论

CLI-Manager 当前的基础能力已经比较完整，主要价值集中在三件事：

1. **把多项目终端集中起来**：项目树、内置终端、分屏、外部终端。
2. **把 Claude / Codex 使用过程管理起来**：Hook 状态、通知、历史会话、Diff、Prompt。
3. **把长期使用数据沉淀下来**：命令模板、统计看板、版本化 WebDAV 备份。

后续最自然的增量方向不是继续堆普通终端功能，而是围绕 README Roadmap 做：

- Claude 配置同步增强
- 项目启动 Profile / Run Profile
